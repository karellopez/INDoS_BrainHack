June 12 Deeper Traintrack 1.5 hours

Multimodal raw-to-BIDS curation with BIDS Manager: concepts, workflow, and practice

Organizer-ready short description: An extended practical introduction to multimodal raw-to-BIDS curation with BIDS Manager, combining BIDS concepts, pre-conversion acquisition inspection, metadata decisions, and a software-guided workflow for organizing MRI, MEG, and EEG data into reusable BIDS datasets.

Most BIDS workshops explain the standard. This session shows how to actually curate real raw neuroimaging data with BIDS Manager: an interactive software tool that lets users inspect how a dataset was acquired, how it is organized, and what problems need attention before conversion.

Description

Raw-to-BIDS conversion is often presented as a naming problem, but in practice it is a curation problem. Real datasets contain repeated acquisitions, inconsistent folder structures, ambiguous sessions, acquisition-time differences, scanner-specific sequence names, missing or confusing metadata, and modality-specific details that are difficult to track manually.

Before conversion, BIDS Manager scans the raw data and presents an interactive view of the dataset: subjects, studies, sessions, source folders, sequence names, acquisition times, file counts, series identifiers, repeated acquisitions, inferred modalities, and proposed BIDS names.

The 1.5-hour format allows time to discuss why curation decisions matter, how metadata affects downstream analysis, and how a curated BIDS dataset becomes a stronger foundation for validation, sharing, analysis, and quality control.

Learning objectives

  • Understand why raw-to-BIDS conversion is a curation workflow, not only a file-renaming step.
  • Learn how BIDS Manager reveals dataset organization and acquisition structure before conversion.
  • See how interactive inspection can catch acquisition and organization problems earlier than manual workflows.
  • Understand how MRI, MEG, and EEG data can be approached within one BIDS curation framework.
  • Connect pre-conversion curation decisions to validation, sharing, analysis, and quality control.

Target audience: Beginner to intermediate participants who want a practical, software-supported route into reproducible neuroimaging data curation.

Raw-to-BIDS curation software

BIDS Manager shows the dataset before conversion

To our knowledge, BIDS Manager is the only interactive raw-to-BIDS tool that gives users this kind of pre-conversion acquisition overview for BIDS curation. It is a curation environment where users can inspect the raw dataset, identify acquisition or organization problems, adjust metadata and naming decisions, preview the future BIDS structure, and then run a more controlled conversion workflow.

01

Acquisition audit before conversion

Review subjects, studies, sessions, source folders, sequence names, series UIDs, acquisition times, repeats, inferred modalities, and file counts before committing to a BIDS structure.

02

Interactive curation, not manual folder editing

Adjust BIDS-relevant names and metadata, apply exclusion rules, adapt suffix patterns to local protocols, detect repeats, and handle mixed-session folders in a guided interface.

03

Preview the future BIDS dataset

Inspect planned BIDS paths in text and tree views before conversion, reducing avoidable mistakes and making decisions easier to explain later.

MEG/EEG quality-control software

MEEGqc turns visual QC into reproducible evidence

MEEGqc is a BIDS-aligned toolbox for reproducible MEG and EEG quality assessment and quality-control reporting. It computes structured quality metrics, generates interactive reports, writes machine-readable derivatives, and supports transparent summaries across recordings and datasets.

01

Structured metrics instead of subjective notes

Compute signal variability, flat or noisy channels, peak-to-peak behavior, PSD, ECG/EOG contamination, muscle artifacts, and MEG head-motion summaries when available.

02

Interactive reports and BIDS derivatives

Generate per-subject HTML reports for inspection and machine-readable outputs for documentation, automation, and downstream processing.

03

Transparent decisions at scale

Use group reports and configurable QC criteria to compare quality evidence across datasets instead of hiding decisions in manual judgement.
